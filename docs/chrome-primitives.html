☿ Chrome Primitives

artboard_03 · frame.w0rk system reference · 5 primitives + 1 assembly

Five repeatable patterns. The chrome is a framing device that makes content feel held, not costumed. If removing it makes the content harder to parse, it's working. If it makes no difference, cut it.

1 · Window Header

--plum (tier 2) · 32px h · pad 8/12
dots 8px: --gold fill + 2 chrome stroke
title: mono 500 · 11px · --muted · +0.03em
border-bottom: 1px --border-subtle

use on: proof cards, project showcases, OS Kit modules

proof_module.exe

2 · Section Frame

1px --border-chrome · radius 4
--panel bg (tier 1) · no titlebar
optional mono label · 12px inset · --muted · uppercase

use on: page sections needing separation without a full window

Section Label

Content sits inside the frame. The border gives the contained feel without the full window treatment: no dots, no title, no status.

3 · Tab Strip

mono 500 · 12px · uppercase · +0.06em
default --muted · hover --cream (no underline)
active --gold + 2px gold underline (offset 4px)
gap 24px (sp-6)

use on: content switching in chrome windows, OS Kit categories

Active Hover Default Default

4 · Status Strip

--ink bg (tier 0) · 24px h · pad 4/12
border-top: 1px --border-subtle
mono 500 · 10px · --muted · space-between
left: glyph + label · right: state / timestamp / progress

use on: bottom of chrome windows, footer-like metadata

1px --border-subtle · full width
A: plain line

glyph --muted or --gold · same size as surrounding text

use on: section breaks, between content groups
